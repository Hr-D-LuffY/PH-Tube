var:
Variables declared with var are function-scoped or globally-scoped, meaning they are only visible within the function where they are declared or globally if declared outside any function. If a variable is declared with var inside a block (like an if statement or a for loop), it is still accessible outside that block.

let:
Variables declared with let are block-scoped, which means they are only accessible within the block where they are declared. They are only accessible after the point of declaration.

const:
Variables declared with const are block-scoped, similar to let. The main difference is that const variables cannot be reassigned after declaration. They are constants.

null use cases:
a programmer can assign this value to indicate intentional absence or no value. and to explicitly clear the value of an object or variable.

undefined use cases:
we can use it as the default value of uninitailized variables and as the default return value of functions that do not have a return statement.
